- 所有路径以 `Path(__file__).resolve().parents[1]` 推导 ROOT，再以 `ROOT / "posts"`、`ROOT / "data"` 等组合常量表达，避免硬编码相对路径。
- 对外部输入（Front Matter 标量）统一走 `parse_scalar` 进行类型推断（字符串/布尔/整数/浮点/逗号分隔列表），保持元数据解析一致性。
- 布尔字段通过 `bool_value(value, default)` 归一化，兼容 true/false/1/0/True/False 多种写法。
- JSON 与 JS 产出成对写入：`write_json(path, payload)` 与 `write_js(path, global_name, payload)` 共用同一目录创建逻辑，JS 产物挂载到 `window.__BLOG_*` 全局变量。
- Markdown 内容处理采用正则流水线：先用 `strip_markdown` 剥离代码块/链接/图片/标题标记，再用 `excerpt_from_body` 按空行分段取首段截断生成摘要。
