本模块采用「HTML 骨架 + 运行时注入」的无构建架构，所有文件平铺于根目录：
- 页面入口：`index.html`(home)、`fragments.html`(fragments)、`article.html`(article) 以及占位页 `works.html`/`shelf.html`/`about.html`，均通过 `<body data-page="...">` 声明页面类型，并由 `script.js` 在启动时按 `data-page` 分支挂载对应逻辑。
- 共享基础设施集中在 `script.js`：提供 `loadDataScript` 动态注入 `data/posts.js`、`data/categories.js`、`data/fragments.js` 等数据脚本（以全局变量 `__BLOG_POSTS__`/`__BLOG_CATEGORIES__`/`__BLOG_FRAGMENTS__` 暴露），并导出 `BlogShared` 命名空间供其他脚本复用 `escapeHtml`、`resolveAssetPath`、`sanitizeSegment` 等工具；同时负责动态加载 `header.js`/`footer.js` 并初始化主题切换与导航高亮。
- 可复用 UI 片段：`header.js` 与 `footer.js` 使用 IIFE 自执行，通过 `<template>` 生成 DOM 后替换或插入 `.page-shell` 中的 `[data-shared-header]` / `.site-footer` 占位节点，实现跨页面共享。
- 文章详情：`article.js` 依赖 `BlogShared`，根据 URL 查询参数 `?category=&slug=` 解析出安全化的路径，再调用 `loadDataScript` 拉取 `data/articles/<category>/<slug>.js`，内置一个轻量 Markdown 解析器（段落、列表、引用、代码块、内联语法）完成内容渲染。
- 样式分层：`styles.css` 为全站基础样式，`article.css` 仅被 `article.html` 引入，用于文章详情页专用排版。
- 依赖方向：HTML → script.js → header.js/footer.js/data/*.js；article.js 反向依赖 script.js 导出的 `BlogShared`，不存在循环依赖。
