也许应该写在“关于我”部分，但我更想把它放在首页。对我来说，博客不只是一个陈列结论的地方，更像是一张长期摊开的书桌，允许阅读、学习与生活留下轻微却真实的回声。

## 这里会记录什么

- 阅读后的批注与回望
- 做项目时留下的思路与踩坑
- 一些不适合发成长文、但值得被记住的瞬间

很多内容未必成熟，甚至可能带着阶段性的偏见。但正因为它们处在形成过程中，才更值得被认真存放。

## 为什么改成 Markdown

以前每一篇文章都要维护单独的 HTML 页面，改一次结构就要重复处理很多模板细节。现在把文章源改成 Markdown 之后，编辑会更轻一些：

1. 文章正文只需要专注内容本身。
2. 首页归档和文章详情使用同一份元数据。
3. 后续想新增文章，只需要在 `posts/default/` 下面直接新增一个 `.md` 文件。
4. 如果文章里要插图，就把图片放到 `image/default/about-site/` 下面，在 Markdown 中直接写相对路径即可。

如果你是从首页点进来的，也可以顺手看看这篇 [博客｜序（新）](blog-intro.md)。
